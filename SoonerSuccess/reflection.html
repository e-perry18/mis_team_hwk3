Tailwind CSS Reflection

This site was built using Tailwind CSS utility classes to handle layout, colors, spacing, and responsive behavior directly in the HTML.

Tailwind Features Used

  • Flexbox layout: The header and footer use flex and justify-between to align brand text and navigation links.
  • Grid layout: Team member cards and importance reasons use grid with sm:grid-cols-2 and lg:grid-cols-4 to adapt to different screen sizes.
  • Responsive breakpoints: Classes like md:hidden, sm:px-6, and lg:px-8 adjust spacing and visibility across mobile, tablet, and desktop.
  • Hover states: Navigation links and cards use hover:text-yellow-200 and hover:shadow-md to provide interactive feedback.
  • Colors, borders, and shadows: Backgrounds like bg-red-700 and bg-slate-50, borders such as border-l-4, and shadows like shadow-sm create visual hierarchy.
  • Typography utilities: Classes including text-3xl, font-extrabold, and tracking-wide help size and style text consistently.

Tailwind vs. Bootstrap (Group Opinion)

For our project, we used Tailwind CSS to build the Sooner Success website and found it more flexible than Bootstrap. Instead of relying on pre-built components, Tailwind uses small utility classes (like flex, grid, and md:), which helped us control spacing, colors, and responsiveness directly in the HTML. At first the class names were confusing, and the HTML looked long, but once we got used to it, building layouts felt faster and more customizable than adjusting Bootstrap defaults. We especially liked how easy it was to make the site responsive using Tailwind breakpoints. Overall, we would rate Tailwind 8.5/10 and recommend it for projects where you want a clean, unique design, while Bootstrap is still better for quick, standard layouts.
